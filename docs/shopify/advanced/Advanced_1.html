按需加载所需的样式（CSS 文件）

{%- unless skip_styles -%}
  {{ 'component-rating.css' | asset_url | stylesheet_tag }}
  {{ 'component-volume-pricing.css' | asset_url | stylesheet_tag }}

  {{ 'component-price.css' | asset_url | stylesheet_tag }}
  {{ 'quick-order-list.css' | asset_url | stylesheet_tag }}
  {{ 'quantity-popover.css' | asset_url | stylesheet_tag }}
{%- endunless -%}

skip_styles 是外部传入的参数，它默认是 nil

✅ {{ 'xxx.css' | asset_url | stylesheet_tag }}

这一行会：

  1. 将 xxx.css 的路径转成完整的 URL（例如 /assets/component-price.css）
  2. 包装成 HTML 的 <link> 标签形式

例如：

{{ 'component-price.css' | asset_url | stylesheet_tag }}

最终生成：

<link
  href="/assets/component-price.css"
  rel="stylesheet"
  type="text/css"
  media="all"
/>

这些文件通常都在 assets/ 目录下，并由 Dawn 或其它主题结构预先定义。

自定义 CSS 也要这样调用。

{{ 'selfdefined.css' | asset_url | stylesheet_tag }}

"t:" 前缀

"name": "t:sections.main-product.name" 在 Theme Editor 中显示的名称，支持多语言 "t:" 前缀表示引用翻译 JSON 中的字符串.

关于templates/index.json

这个文件是对主页的修改部分，并且在里面配置一些记录的参数。

关于在商品列表中常见的设置

{
  "type": "product_list",
  "id": "product_list",
  "label": "Product list"
}

这个是 Shopify 主题编辑器支持的新功能，允许用户选择多个产品 所选的产品信息会保存在 settings.product_list 中，你可以通过 section.settings.product_list 访问 它不对应本地文件，不需要你手动配置数据源。

关于动态显示产品的时候 collection 和 Product list

✅ Shopify Theme Editor 中的表现：

状态显示内容逻辑
Product list 不为空只显示 product_list 中手动选择的产品
Product list 为空，Collection 被选择了显示 collection 中的产品
两个都为空不显示任何产品
